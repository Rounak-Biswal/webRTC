# 3️⃣ Should server store current call? (VERY good question)

👉 **YES — 100% yes**

You’re thinking correctly.

---

## Why you need it

Right now:

> B accepts → server blindly trusts it

But what if:

* A never called B?
* malicious client sends fake accept?

👉 Your system breaks

---

## What to store (simple)

Maintain:

```text
activeCalls = {
  callerSocketId: calleeSocketId
}
```

---

## Flow with validation

### On call request:

* store:

  ```
  caller → callee
  ```

---

### On accept:

* verify:

  ```
  does caller → callee exist?
  ```

If not:
❌ reject event

---

## Bonus benefit

Later you can:

* prevent multiple calls
* handle busy users
* implement call timeout
